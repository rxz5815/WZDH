# 网站导航系统

这是一个简洁、可自定义的网址导航页面，纯静态，支持Cloudflare搭建，喜欢折腾的可以二次开发。


## 功能特点

- 响应式设计,适配各种设备
- 支持在线添加和编辑站点链接
- 支持删除单站点与分类（删除分类会删除分类下的所有站点）
- 支持分类下创建子分类（删除子分类不会删除站点）
- 密码保护的编辑功能
- 添加站点时自动抓取网站图标
- 右键站点图标可直接编辑站点信息
- 带站内搜索链接
- 带站外搜索功能
- 支持添加分类/删除分类
- 支持拖拽站点到别的分类
- 支持手动拖拽站点排序
- 支持手动拖拽分类/子分类排序
- 自带一键更换背景渐变色/风景图
（刷新页面会保持原背景，只有清除浏览器缓存才会恢复默认背景）
- 自动核对重复链接
（建同一站点，只要后缀不同即可，一模一样的站点会默认进入编辑模式）
- 基于Cloudflare Pages,部署简单,访问快速

## 部署步骤

### 1. 复刻仓库


### 2. 创建Cloudflare Pages项目

1. 登录[Cloudflare Dashboard](https://dash.cloudflare.com/)（进入网站后，选简体中文）
2. 点击“计算”
3. 点击“Workers 和 Pages”
4. 点击“创建应用程序”
5. 点击“想要部署 Pages？开始使用”
6. 点击“导入现有 Git 存储库”，点开始使用
7. 选择"连接到Git",然后选择您的GitHub仓库（需要登入账号，并复刻项目）
8. 选择复刻的项目-点开始设置（名称随便取，分支默认）
9. 在构建设置中:
   - 构建命令: 留空
   - 构建输出目录: 留空
10. 点击"保存并部署"

### 3. 设置KV命名空间

1. 在左侧中,点击"存储和数据库" > "Workers KV"
2. 点击"创建命名空间"
3. 命名为"LINKS_KV"

### 4. 绑定KV到Pages项目

1. 回到Pages项目设置中,点击绑定（定义您名下 Pages Functions 可用的资源集）
2. 选择"KV命名空间绑定"
3. 点击"添加绑定"
4. 变量名称设为"LINKS_KV",选择刚才创建的KV命名空间

### 5. 配置环境变量

1. 点击“变量和机密”
2. 添加以下变量:
   - `NODE_VERSION`: 设置为`18`或更高版本
   - `EDIT_PASSWORD`: 设置为您选择的编辑密码

### 6. 重新部署

回到项目"部署"页面,点击"重新部署"以应用新的设置。

## 使用说明

### 访问导航页面

部署完成后,您可以通过Cloudflare Pages提供的URL或设置的自定义域名访问导航页面。
